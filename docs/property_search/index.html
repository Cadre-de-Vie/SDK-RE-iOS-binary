L'API permet de rechercher les properties correspondant à des critères géographiques, ou des critères de la property (surface, nombre de pièces...)

Critères de recherche

Pour effectuer une recheche, il faut créer un objet de type CDVPropertySearch et lui définir les attributs à rechercher. Les détails des properties sont décrites dans CDVPropertySearch.h Il est possible de préciser une recheche avec des critères géographiques en utilisant : des cercles (cdvGeoDistance) : une distance en kilomètres autour d'un point central en coordonnées GPS (lat,lng). Voir ci-dessous l'autre usage de cdvGeoDistances en tant que filtre des résultats et non plus critère de recherche ; des polygones (cdvGeoPolygons) : au moins 3 points en coordonnées GPS (lat,lng). Il est possible de limiter les résultats retournés à ceux visibles dans le viewport d'une carte : * viewport (cdvGeoBoundingBox) : obligatoirement caractérisé par ses coordonnées GPS, d'abord Nord-Est (lat,lng) puis Sud-Ouest (lat,lng).

Il est possible de limiter une recheche sur des critères géographiques en utilisant des polygones, une distance (en kilomètres) autour d'un point, ou une bounding box

 let propertiesSearch = CDVPropertySearch.init()

 propertiesSearch.cdvPageSize = 1
 propertiesSearch.cdvSort = .none
 propertiesSearch.cdvService = CDVPropertyService.buy
 propertiesSearch.cdvParentTypes = ["house", "apartment"]
 propertiesSearch.cdvNbRooms = [2,3]
 propertiesSearch.cdvMinPrice = 600000
 propertiesSearch.cdvMaxPrice = 99000

 propertiesSearch.cdvGeoDistances = [CDVGeoDistance(dictionary: ["lat": 48.8834611,
                                                                 "lng": 2.2652248,
                                                                 "distance": 2])]
 propertiesSearch.cdvGeoPolygons = [CDVGeoPolygon(array: [
     [
         "lat": 45.755225,
         "lng": 4.841869
     ],
     [
         "lat": 45.751333,
         "lng": 4.854121
     ],
     [
         "lat": 45.762097,
         "lng": 4.858005
     ],
     [
         "lat": 45.759567,
         "lng": 4.847148
     ],
     [
         "lat": 45.755225,
         "lng": 4.841869
     ]
 ])];

 propertiesSearch.cdvGeoBoundingBox = CDVGeoBoundingBox(array:
     [
         [
             "lng": -7.237292528124954,
             "lat": 52.36947753602144
         ],
         [
             "lng": 11.637219190625046,
             "lat": 40.036761141167766
         ]
 ]);

Recherche pour obtenir une liste

Une fois l'objet CDVPropertySearch créé il est possible de recherche les properties correspondantes


 CDVManager.sharedInstance().getPropertiesWith(propertiesSearch) { (error, properties, totalProperties) in
if (error != nil) {
    NSLog("getPropertiesWithSearch Error: \(error!.localizedDescription)")
} else {
    NSLog("\(totalProperties) properties found with this search but \(properties!.count) received \(String(describing: properties))")
    for details in properties!
    {
        NSLog("Property details\(String(describing: details))")
    }
}}

Pour obtenir les résultats suivants (c-a-d la page suivante) il faut incrémenter la property cdvPage de l'objet CDVPropertySearch La première page a l'index 1, la seconde page a l'index 2, etc-etc...


     propertiesSearch.cdvPage = 2
 CDVManager.sharedInstance().getPropertiesWith(propertiesSearch) { (error, properties, totalProperties) in
if (error != nil) {
    NSLog("getPropertiesWithSearch Error: \(error!.localizedDescription)")
} else {
    NSLog("\(totalProperties) properties found with this search but \(properties!.count) received \(String(describing: properties))")
    for details in properties!
    {
        NSLog("Property details\(String(describing: details))")
    }
}}

Geo Precision (paramétrage)

The GeoHashgrid precision can either be defined in terms of the integer precision levels mentioned in the Elastic Search documentation. Values outside of [1,12] will be rejected. Alternatively, the precision level can be approximated from a distance measure like "1km", "10m" (cf. https://www.elastic.co/guide/en/elasticsearch/reference/current/search-aggregations-bucket-geohashgrid-aggregation.html)

La précsion du regroupement par cluster, maillage ou fonction GeoHashgrid, définie de 3 façons : la valeur zéro "0" : indique que le calcul des clusters doit être désactivé pour renvoyer la liste de tous les points individuels, chacun dans un pseudo-cluster une valeur entière, de [1,9] (ne pas utiliser 10,11 et 12) : indique que le calcul des clusters est activé pour regrouper les points dans des clusters d'une largeur/hauter telles que définies dans la documentation de Elastic (cf. https://www.elastic.co/guide/en/elasticsearch/reference/current/search-aggregations-bucket-geohashgrid-aggregation.html) une distance reconnue par Elastic telle que "1km","100m" (voir la documentation officielle) : indique que le calcul des clusters est activé pour regrouper les points dans des clusters de cette largeur/hauteur (e.g. 1km x 1km) toute autre valeur entière ou réelle au-delà de ces bornes sera ignorée

propertiesSearch.cdvGeoPrecision = "2"

/* 3 types of possible values (String):
* - zero: '0'
* - GeoHashgrid code: '1', '2', ..., '9'
* - Distances: '1m', '10km'
*/

Plusieurs options sont disponibles lors d'une recherche de type "Carte":

  • cdvHideClusterRect (Boolean) - Ne calcule pas et ne renvoie pas les champs "minlat", "minlng", "maxlat" et "maxlng" décrivant le rectangle géographique dans lequel tous les points d'un cluster sont contenus
  • cdvHidePropertyIds (Boolean) - Ne renvoie pas la liste des id "propertiesIds"
  • cdvHideSinglePoint (Boolean) - Ne calcule pas et ne renvoie pas le champ "singlePoint" précisant si le cluster peut être éclaté en augmentant la valeur de Geo Precision (cluster "éclatable" limité à 5m x 5m, donc cdvGeoPrecision = 9)

Recherche du nombre de properties

Cette recherche permet d'obtenir uniquement le nombre de properties correspondant à une recherche, et non détails sur les properties

CDVManager.sharedInstance().countProperties(with: propertiesSearch) { (error, totalProperties) in
            if (error != nil) {
                NSLog("countPropertiesWithSearch Error: \(error!.localizedDescription)")
            } else {
                NSLog("\(totalProperties) properties found with this search")
            }
        }

Tri des propriétés


propertiesSearch.cdvSort = .none

/* Possible values (CDVPropertySort):
 * .none
 * .new
 * .priceAsc
 * .priceDesc
 * .pricePerMeterAsc
 * .pricePerMeterDesc
 * .surfaceAsc
 * .surfaceDesc
 * .distanceAsc
 * .distanceDesc
 */

Note importante : le tri par distance croissante/décroissante (.distanceAsc, .distanceDesc) requiert le paramétrage de cdvGeoDistances avec un seul cdvGeoDistance pour définir le point de départ du tri et sa distance. Cela permet par exemple de "lister les properties jusqu'à 500m de ma position", en paramétrant la position GPS de l'utilisateur comme le centre de cdvGeoDistance et une distance de 0.5 km, et en paramétrant cdvSort = .distanceAsc

Parent types (Type des propriétés)

propertiesSearch.cdvParentTypes = ["house", "apartment"]

/* Possible values (Array of Strings):
 * - 'apartment'
 * - 'local'
 * - 'house'
 * - 'field'
 * - 'parking'
 * - 'other'
 */
